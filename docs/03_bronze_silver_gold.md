# 03 Bronze / Silver / Gold

## 目的

生データを直接分析せず、加工責務を層で分離して品質と再利用性を高めます。

## 各層の役割

| 層 | 役割 |
| --- | --- |
| Bronze | S3の生データをそのまま取り込む |
| Silver | 型変換、NULL除去、不要列削除 |
| Gold | 集計済みデータ、分析用テーブル |

## 変換の基本ルール

- Bronze: 取り込み優先（できるだけ原本保持）
- Silver: データ品質の担保（型・欠損・重複・命名）
- Gold: KPIやダッシュボード利用に最適化

## 説明例

> 生データを直接分析せず、Bronze / Silver / Goldの3層に分けて、加工履歴と用途を整理しました。

## 実装メモ

- Delta Table形式で保存
- 層ごとにテーブル名を明確化（例: bronze_sales, silver_sales, gold_sales_daily）
- 冪等実行を意識（再実行しても破綻しない処理）
